# Toolbox Design System

> 状态：Draft v0.2 · 规范性文档
>
> 本文定义所有 Toolbox 工具必须共享的视觉与交互基础。工具可以保留自己的信息架构和个性，但不能另造全局主题、语言或导航行为。

文中的“必须”是发布门禁，“建议”允许在有明确理由时偏离。当前代码尚未全部满足本文，差距记录在 [TASKS.md](./TASKS.md)。

## 一、设计原则

1. **先清楚，再装饰**：结果、单位、输入和假设必须一眼可辨。
2. **一致的壳，自由的工具**：全局导航、偏好、token 与交互统一；业务布局按问题选择。
3. **明暗等价**：浅色不是深色的反相，两个主题都必须具备相同层级和可读性。
4. **双语不降级**：中文和英文都不能出现截断、错位或缺失。
5. **不以鼠标为前提**：键盘、触屏、减少动态效果和窄屏均需可用。
6. **系统字体与本地资产优先**：不静默加载远端字体或装饰资源。

## 二、全局应用壳

每个工具页面由两层组成：

```text
Toolbox Shell（所有工具一致）
  └─ NavBar：工具切换、语言、主题、移动菜单

App Surface（工具自己拥有）
  ├─ 工具标题与简介
  ├─ 业务导航或模式切换
  ├─ 输入 / 结果 / 解释
  └─ 工具页脚
```

- 全局 NavBar 必须是页面上唯一的主题和语言入口。
- 工具可以有自己的 tab、sidebar 或步骤导航，但不能重复全局偏好按钮。
- 全局壳高度、左右留白和窄屏折叠行为必须由共享实现控制。
- App Surface 不应覆盖 `.toolbox-nav*` 样式，也不应修改全局 `html` / `body` token 的语义。

## 三、主题契约

### 3.1 模式与首次选择

| 模式 | 色板 | 行为 |
|------|------|------|
| dark | Catppuccin Frappé | 深色主题 |
| light | Catppuccin Latte | 浅色主题 |

- 已保存偏好优先读取 `localStorage['toolbox-theme']`。
- 未保存时遵循 `prefers-color-scheme`；若系统信息不可用则回退 dark。
- `<html data-theme="dark|light">` 必须在首屏绘制前设置，避免闪烁。
- 切换主题不得清空工具业务状态。

### 3.2 Token 分层

应用应消费语义 token，而不是复制色号：

```text
原始色板   --ctp-blue / --ctp-base / --ctp-text
    ↓
语义色     --color-bg / --color-text / --color-border / --color-danger
    ↓
组件角色   --button-bg / --result-accent（仅在确有稳定语义时增加）
```

必须共享的 token 类别：

- 背景：页面、浮层、卡片、弱化区域。
- 文字：主要、次要、弱化、反色。
- 边框与焦点：默认、强调、focus ring。
- 状态：primary、success、warning、danger、info。
- 字体：sans、mono、数字的 tabular figures。
- 形状：`8 / 12 / 18 / 22px` 圆角与 pill。
- 动效：快速反馈、主题过渡、减少动态效果。

应用专属色可以存在，但必须在 light/dark 下分别验证对比度，且不能重新定义全局 token。

## 四、导航与右上角控件

### 4.1 导航结构

- 左侧：Toolbox 品牌与完整工具下拉菜单。
- 中部：桌面快速入口；当前工具可高亮。
- 右侧：语言、主题；窄屏再显示菜单按钮。
- 下拉菜单必须同时支持 hover、keyboard focus 与 tap。

### 4.2 语言与主题按钮（已确认偏好）

语言与主题按钮必须遵循同一交互：

- 默认状态无独立底色、边框或常驻外框。
- 鼠标 hover **不出现背景块、描边或类似“选中框”的容器**。
- hover 可以通过图标/文字颜色、透明度或轻微图标动效反馈。
- active 可以有短暂缩放，但不能导致布局位移。
- 键盘 `:focus-visible` 必须保留清晰的 2px focus ring；“无 hover 选中框”不等于移除键盘焦点。
- 图标按钮的可点击区域至少 `36 × 36px`，触屏目标建议达到 `44 × 44px`。
- 语言按钮展示“将切换到的语言”，并具有双语 `aria-label`。
- 主题按钮的图标、标题与真实当前主题一致。

当前共享 CSS 对 `.toolbox-nav-icon-btn:hover` 设置了背景，且 `focus-visible` 被清空；这是待修复项，不是规范本身。

## 五、排版与密度

### 字体

```css
--font-sans: ui-rounded, "SF Pro Rounded", system-ui,
  "PingFang SC", "Noto Sans SC", sans-serif;
--font-mono: "JetBrains Mono", "SF Mono", ui-monospace, monospace;
```

- 结果数字、价格、日期、尺寸和单位建议使用 mono + tabular figures。
- 正文不小于 14px；辅助信息不建议低于 12px。
- 标题层级由语义决定，不通过跳级获得视觉大小。
- 中文与英文使用同一视觉层级，但允许针对英文长词调整容器而不是缩小字号。

### 间距

建议以 4px 为基准：`4 / 8 / 12 / 16 / 24 / 32 / 48 / 64`。

- 同组控件使用 8–12px。
- 卡片内部主要区块使用 16–24px。
- 页面主要区块使用 24–48px。
- 不为单个页面随意引入接近但不同的间距值。

### 信息密度

- 输入与结果应在首个主要视口内建立因果关系。
- 解释、假设和术语可以折叠，但不能隐藏影响结论的关键条件。
- 装饰性卡片不能比核心结果更抢眼。

## 六、基础交互状态

所有可交互控件必须定义：

| 状态 | 要求 |
|------|------|
| default | 对比度和可点击性清楚 |
| hover | 只作为鼠标增强，不承载唯一信息 |
| active | 即时反馈，无布局跳动 |
| focus-visible | 明确焦点环，不得全局 `outline: none` |
| disabled | 视觉弱化并使用原生 disabled 语义 |
| loading | 保留控件尺寸，说明正在处理 |
| error | 说明原因与恢复方式，不只变红 |

`prefers-reduced-motion: reduce` 下必须关闭旋转、位移和大范围过渡，只保留必要状态变化。

## 七、语言契约

- 支持 `zh` 与 `en`，HTML 分别写入 `zh-CN` 与 `en`。
- 全局语言状态使用 `toolbox-lang`，切换后所有可见区域同步更新。
- 工具文案归工具维护；全局导航文案归共享层维护。
- 不在 JSX/JS 中拼接依赖语序的句子；使用带参数的完整翻译模板。
- 翻译 key 缺失必须在测试或开发模式可见，不能静默显示空白。
- 日期、数字、货币和百分比使用当前语言的 `Intl` 格式化。
- 页面 `<title>`、description 和 aria 文案也必须随语言切换。

## 八、响应式与内容宽度

最低支持宽度为 375px，关键检查宽度为：

| 宽度 | 目的 |
|------|------|
| 375px | 手机最小宽度 |
| 768px | 平板 / 大手机横屏 |
| 1024px | 小笔记本 |
| 1440px | 桌面 |

- 不允许全页横向滚动；数据表可在自己的容器内滚动。
- 移动端不能只靠 hover 打开菜单或提示。
- 业务 sidebar 在窄屏应转换为 tab/drawer，而不是压缩内容。
- 结果与单位不可因窄屏被拆得难以理解。

## 九、无障碍底线

- 使用语义 HTML；自定义 tab、dialog、menu 必须具备对应键盘行为。
- 所有输入都有关联 label，错误信息可被辅助技术读取。
- 所有 icon-only button 都有可本地化的 accessible name。
- 文字与背景达到 WCAG AA；状态不能只靠颜色区分。
- Canvas 图表提供文本摘要或等价数据说明。
- 页面切换或计算完成后的焦点与 live region 行为应可预测。

## 十、允许的个性与禁止的分叉

允许：

- 与工具领域相关的图标、图表、业务布局和强调色。
- RateLens 的步骤流、ChronoSphere 的 tabs、Monitor Choice 的实验室结构、SaneUnits 的多计算器导航。
- 不同的信息密度，只要 token、交互与响应式契约一致。

禁止：

- 第二套全局主题或语言按钮。
- 自定义全局 storage key、色板命名或明暗解析规则。
- 复制并手工修改共享导航代码。
- 用页面级 CSS 覆盖共享壳实现。
- 仅在某个工具出现的外部字体、追踪脚本或无说明网络请求。

## 十一、当前差距

| 区域 | 差距 | 目标 |
|------|------|------|
| Theme | `@toolbox/theme` 未被应用直接消费 | 逐个应用迁移到语义 token，并有视觉回归 |
| Homepage / Monitor | nav/theme 是手工副本 | 使用构建期依赖或自动生成且校验 hash |
| Nav actions | hover 有背景；focus-visible 无可见 ring | 按第 4.2 节统一 |
| SaneUnits | 共享 NavBar 外还有两套响应式偏好控件 | 只保留共享入口，业务导航保持自身特色 |
| Visual QA | 没有跨主题/语言/视口矩阵 | CI 生成 smoke 截图并做人工/自动审查 |

## 十二、发布前设计检查

- [ ] light / dark 下无不可读或层级消失。
- [ ] zh / en 下无截断、空白 key 或布局溢出。
- [ ] 375 / 768 / 1024 / 1440px 下关键流程可用。
- [ ] 语言与主题只在共享 NavBar 出现一次。
- [ ] 右上角控件 hover 无选中框，focus-visible 有焦点环。
- [ ] 键盘可完成主流程，触屏不依赖 hover。
- [ ] 无新增远端字体、追踪或未披露的外部请求。
- [ ] 与稳定版本的视觉差异有意且在 PR 中说明。
