# FormTran

一个纯浏览器、本地处理的文件工作台，覆盖转换、压缩、编辑、拆分、合并、编码、解析和信息查看。文件首页先识别输入并推荐相关工具，用户明确选择后才开始处理。任何源内容都不会离开设备。

扩展按阶段交付：图片转换（含 HEIC/HEIF 本地解码）、GIF 合成、文本/标记转换、CSV/TSV/XLSX 表格转换、PDF 轻量检查和有边界的 ZIP 清单/解压已可用；图片编辑、GIF 专项处理、PDF 页面工具和其他压缩格式仍须通过各自的正确性与资源边界后才能开放。“可识别”不代表当前浏览器一定能完整解码或编辑。详细约束见[文件工作台架构](./docs/FILE_WORKBENCH.md)。

## 文件首页

- 支持混合文件和文件夹，队列上限为 500 个文件、总计 2 GB。
- 每个文件最多读取前 64 KiB，用于识别 JPEG、PNG、WebP、AVIF、GIF、SVG、BMP、TIFF、HEIC/HEIF、ICO、PDF、ZIP，以及受支持的文本和结构化数据文件名。
- 能识别内容签名时优先使用签名；内容与扩展名冲突会明确警告。
- 会区分 XLSX、DOCX、ODS、EPUB 等以 ZIP 为容器的文档格式与普通 ZIP，避免错误推荐压缩包解压工具。
- 分开标记可用、有限支持和规划中能力；不会自动启动转换或解析。
- 支持把兼容输入带入图片转换、GIF 合成、文本/标记转换、表格数据、PDF 检查或 ZIP 检查；10 MB 以内图片可在本地生成 Data URL。
- 图片提供网页照片、透明素材和隐私分享三个可继续编辑的起始预设。

## 输入与输出

- 输入：队列最多 500 张，单文件最多 512 MB，总计最多 2 GB；HEIC/HEIF 另设 64 MB 解码上限。
- 输出：PNG（无损、透明）、JPEG（有损、不透明）或 WebP（有损、透明）。
- HEIC/HEIF 先尝试浏览器原生解码，失败后按需加载本地 Apache-2.0 WASM 解码器；该约 960 KB 原始体积的模块不会进入首页首包，也不会从第三方 CDN 加载。
- 尺寸：保持原图、百分比缩放或限制最大宽高；可禁止放大小图。
- 变换：支持对队列批量旋转 0°/90°/180°/270°，并按最终输出方向水平或垂直翻转。
- 图片信息：对于可安全预览的位图，转换前即可在队列查看解码像素尺寸、像素总数、宽高比、浏览器 MIME 类型和文件大小。
- 文件夹导入会在按需展开的明细中列出每个未加入文件及原因，不会静默忽略。
- 命名：可点击变量模板或带常用预设的引导式正则替换，提供捕获组反馈、逐文件命中状态、实时预览、补零序号、跨平台文件名清理和自动重名后缀。
- 文件夹上传可以在 ZIP 中保留相对目录结构。
- 转换结果进入图库，可通过键盘操作的弹窗查看转换前后对比。
- 下载可选择直接文件（单张直接下载、多张逐个下载）或一个 ZIP 压缩包。
- 上传区和文件队列位于工作台首行，每次选择后立即显示成功与未加入数量。
- 独立「知识库」Tab 按图片、动画、文本标记、PDF 和压缩包提供能力对比与安全边界。

## GIF 合成

- 支持 2–100 张浏览器可解码图片，可显式调整帧顺序。
- 可清空整个帧队列，也可逐帧移除和排序。
- 所有帧等比适配到统一画布，可设置尺寸、帧延迟、循环次数和留白底色。
- 在浏览器本地编码 GIF89a，完成后直接预览和下载。
- 零依赖编码器使用固定 3-3-2、256 色调色板，适合简单短动画；照片和渐变可能出现色带。
- 单边限制 4096 像素，总工作量限制 1 亿帧像素，降低页面崩溃风险。

## Text & Markup 转换

- 可打开或粘贴 TXT、Markdown、Org mode、reStructuredText、AsciiDoc 与 HTML。
- 最多可打开 100 个文本文件进入可见批量队列，支持切换、逐项移除、清空，并把全部转换结果打包为 ZIP。
- 通过共享块结构解析标题、段落、列表、引用、代码块、分隔线和常见链接，再生成任一目标格式。
- 可检查识别到的结构，预览原始输出或受限 HTML，复制或按对应扩展名下载。
- HTML 会移除危险元素，不保留危险链接；预览运行在禁止脚本的 sandbox iframe 中。
- 方言扩展会被有意规范化；往返转换保留已支持的结构，不保证源文本逐字一致。
- 源/目标格式与图片下载方案均使用支持键盘操作的主题原生菜单，不依赖浏览器差异较大的原生下拉渲染。

知识库使用用途决策行、可展开格式说明和图片/动画/标记语言对比表，不再把所有知识点表现成同层级 Card。

## 表格数据转换

- CSV/TSV 按 UTF-8、标准引号与自动分隔符识别读取；XLSX 按工作簿结构读取，不会被当成普通 ZIP 解压。
- CSV/TSV 可导出 CSV、TSV、JSON 或一个无宏的单工作表 XLSX；XLSX 可导出 CSV、TSV 或 JSON。
- XLSX 只读取单元格值与文件内公式缓存值，不执行公式、宏或外部链接，也不承诺保留日期格式、样式、图表、合并单元格或公式。
- CSV/TSV 导出默认防御表格公式注入；用户可以明确关闭。JSON 可选择把第一行作为经去重的字段名。
- CSV/TSV 最大 16 MB，XLSX 最大 32 MB；最多 20,000 行、256 列、250,000 个单元格，单元格最长 32,767 字符。预览只显示前 50 行、20 列，导出使用全部已接受数据。

## PDF 与 ZIP 工作台

- 可直接进入 PDF、压缩包 Tab，也可从文件首页识别结果主动流转。
- PDF 检查显示版本、轻量页数/对象数估算、加密、线性化、元数据提示和首个页面框；最多读取 32 MB，并明确不冒充完整解析器。
- ZIP 检查限制为 5000 个条目、压缩包 512 MB、单条目展开 256 MB、总展开 1 GB。
- 选择性解压会拦截危险或重名路径、加密、未知算法、超大条目和异常压缩比；Deflate 一旦超过声明大小就立即停止，选中项按顺序展开，并在下载前复核最终大小与 CRC。多项结果在本地重新打包。

## 隐私与联网行为

文件、识别头部、图片、预览、转换结果、解析文本、表格和 ZIP 全部只存在浏览器内存，不会上传。应用没有账号、后端、业务网络请求、遥测、广告、Cookie、追踪或远端字体。只有转换偏好会写入 `localStorage` 的 `toolbox.image-converter.settings`；文件内容、文件名和识别结果不会持久保存。

隐私说明位于当前应用 Tab 的底部，图片、GIF、文本、PDF、压缩包和知识库页面分别说明各自真实的本地数据流。

静态资源可用后，本工具可离线工作。浏览器不支持的编解码会只让对应文件失败，不会修改原文件。

## 重要边界

- 图片格式转换 Tab 会把 GIF、动态 WebP 和动态 AVIF 压平为浏览器解码的第一帧；如需新建 GIF，可使用独立 GIF 合成 Tab。
- Canvas 重编码通常会移除 EXIF、GPS、相机信息和大多数嵌入色彩配置。
- HDR、CMYK、广色域和高位深素材可能被映射到普通浏览器画布色域。
- JPEG 不支持透明，透明区域会铺上用户选择的底色。
- SVG 在解码前会移除脚本、事件、嵌入 HTML 和外部引用；安全清理前不显示原始 SVG 预览。
- 输出最长边限制为 16,384 像素，总像素限制为 8,000 万，降低浏览器内存崩溃风险。
- 原生图片解码能力取决于浏览器版本；HEIC/HEIF 另有随静态资源部署的本地 WASM 后备路径。
- 请求旋转或翻转时一定会重新编码，不会因为原文件更小而返回未变换的原图。
- GIF 合成不读取源动图的全部帧，也不保留逐帧透明、处置方式或源时序。
- 标记转换覆盖实用公共子集，不是完整 CommonMark、Sphinx、Org Babel 或 AsciiDoc 处理器。
- 表格转换以安全、可检查的值转换为目标，不是 Excel 高保真编辑器；不会保留宏、公式逻辑或版式。
