# FormTran 文件工作台架构

FormTran 的产品边界是“浏览器本地文件工具集合”，不再限定为格式转换器。首页接收文件、
识别类型并解释可用操作；真正的转换、编辑或解析仍进入各自独立工作台，避免首页成为控件墙。

## 交互模型

```text
拖入文件 / 选择文件或文件夹
  → 本地识别扩展名、MIME 与有限文件签名
  → 按图片、GIF、PDF、文本与数据、压缩包或未知类型分组
  → 展示文件信息、能力状态和推荐工具
  → 用户明确选择工具后，才读取完整内容并进入对应工作台
```

识别不是安全扫描，也不保证文件内容与扩展名一致。首页只读取分类所需的少量头部字节；
图片像素、完整文本、PDF 页面和压缩包条目只应在用户选择对应工具后按需读取。

## 能力状态

| 状态 | 产品含义 | UI 规则 |
|---|---|---|
| 可用 | 已实现、已测试且当前浏览器具备必要能力 | 可进入工具，说明预设和输出 |
| 有限制 | 已实现基础路径，但编解码或保真度取决于浏览器 | 可进入工具，同时显示具体限制 |
| 规划中 | 已确认产品价值，尚未满足正确性、性能或安全门槛 | 可查看说明，不渲染伪操作按钮 |

任何工具不得仅凭扩展名宣称可处理。运行时能力检测失败时必须保留源文件并给出逐文件原因。

## 文件族与阶段路线

### 图片与 GIF

- 识别：JPG、JPEG、PNG、WebP、AVIF、GIF、SVG、BMP、TIFF、HEIC、ICO。
- 已有基础：PNG/JPEG/WebP 转换、质量与尺寸、批量命名、元数据随 Canvas 重编码清除、
  静态图片合成 GIF。
- 第一阶段推荐入口：转换格式、压缩图片、调整尺寸、查看信息、清除元数据、图片转 Base64、
  多图合成 GIF。
- 后续独立工具：裁剪、旋转与翻转、图片拼接、Favicon 生成、批量水印；GIF 拆帧、压缩、
  调速、裁剪、尺寸、视频和其他动图格式。
- TIFF、HEIC、ICO 和部分 AVIF/BMP 仅保证识别；实际解码取决于浏览器或后续经审核的本地
  编解码器。不得静默上传作降级方案。

### PDF

- 识别 PDF，并推荐合并、拆分、删除/提取/重排/旋转页面、PDF 转图片和信息查看。
- PDF 页面编辑必须保留页面盒、旋转、对象引用和加密边界；没有经过审计的本地解析器前，
  这些能力保持“规划中”。
- 大型 PDF 必须按需读取并允许取消，不能为了显示推荐工具先解析整份文档。

### 文本与数据

- 识别 TXT、Markdown、HTML、JSON、YAML、XML、CSV。
- 已有 TXT/Markdown/HTML 与常用标记格式的批量结构转换。
- 后续入口包括格式化、内容预览、格式转换、文本对比、数据统计和编码转换。
- 数据转换必须明确字符编码、换行、CSV 方言、数字精度、公式注入与 XML/HTML 安全边界；
  大文件优先流式处理。

### 压缩包

- 首批只识别 ZIP；其他压缩格式在具备可审计解析器后逐项加入。
- 查看内容、全部解压、选择性解压和文件列表必须防御路径穿越、重复路径、压缩炸弹、超大
  展开体积和隐藏文件误操作。
- 不以“浏览器可以读 ArrayBuffer”为由直接开放不受限解压。

## 常用预设

预设是可解释的参数组合，不是不可见的魔法：图片提供“网页照片”“透明素材”“社交分享”
和“隐私分享”；GIF 提供“表情循环”“步骤演示”；文本提供“README”“静态网页”“数据检查”。
用户选择预设后仍能看到并修改实际格式、质量、尺寸、时序或解析选项。

## 隐私与开发者信息红线

- 文件名、文件字节、像素、文本、解析结构和输出只存在当前浏览器内存，不写入 storage。
- 只有非内容偏好可使用 `toolbox.image-converter.*`；不使用 Cookie、账号、遥测或指纹。
- 不发起业务网络请求，不加载远端字体、编解码器或运行时资源；不支持的能力明确失败。
- 错误、测试 fixture、README 和交接文档不得包含真实用户内容、个人路径、私有邮箱、主机、
  端口、Token 或设备信息。
- 未来若引入本地 WASM，依赖、许可证、体积、worker 隔离、内存上限和供应链来源必须单独审核。

## 性能与可靠性预算

- 首页单次最多接收 500 个文件、总计 2 GB；默认只读取每个文件最多 64 KiB 用于识别。
- 队列使用分组和渐进信息读取；预览使用 object URL，并在移除或卸载时释放。
- 像素处理继续执行现有最长边和总像素上限；长任务应支持逐项进度与停止。
- PDF、压缩包和大型文本处理进入独立模块前，需要 worker/取消策略和针对恶意输入的测试。
- 新编解码依赖必须记录首包增量；非首页必需能力采用延迟加载，不能拖慢“识别与推荐”。

## 验收原则

1. 同一文件在中英文界面获得一致分类、能力状态和可理解的限制说明。
2. 用户未选择工具前不读取完整文件，也不自动执行转换。
3. 可用操作能把兼容文件带入对应工作台；不兼容文件保持在首页并解释原因。
4. 混合文件、多文件、文件夹、空文件、伪装扩展名、超限输入和未知格式均有确定行为。
5. light/dark、1440/390px、键盘操作、reduced motion、无横向溢出和无业务请求通过回归。
