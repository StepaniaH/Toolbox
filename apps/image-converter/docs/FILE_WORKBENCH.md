# FormTran 文件工作台架构

FormTran 的产品边界是“浏览器本地文件工具集合”，不再限定为格式转换器。首页接收文件、
识别类型并解释可用操作；知道目标的用户也可以直接进入图片、GIF、文本、表格数据、PDF 或压缩包等
文件族 Tab。真正的转换、编辑或解析仍进入各自独立工作台，避免首页成为控件墙。

## 交互模型

```text
拖入任意文件或文件夹 / 展开一个“添加内容”入口
  → 本地识别扩展名、MIME 与有限文件签名
  → 按图片、GIF、PDF、文本与数据、压缩包或未知类型分组
  → 单项、勾选多项、整组或全选；展示实时总览和独立单项预览
  → 展示选择范围、能力状态和推荐工具
  → 用户明确选择工具后，兼容范围才读取完整内容并进入对应工作台
  → 专用工作台保留“只处理这里 / 返回总任务”路径，首页任务不会卸载

或：用户先进入文件族 Tab → 选择该领域工具 → 再导入文件
```

识别不是安全扫描，也不保证文件内容与扩展名一致。首页只读取分类所需的少量头部字节；
安全位图和文本只在用户打开单项预览后读取，PDF 页面和压缩包条目只在用户选择对应工具后按需读取。
浏览器原生文件选择器无法在同一次对话中混选文件与文件夹，因此首页只显示一个“添加内容”控件，
展开后再提供系统级文件/文件夹入口；拖放区域则递归接收两者。这是平台限制，不应重新演变成两个并列主按钮。

共享任务已经覆盖输入、分组、范围、总览、工作台流转与结果回收。各专用工作台只在生成或验证完
新 Blob 后向内存结果注册表发布引用；首页按文件族显示结果，提供单项/勾选/同类/全局范围、逐项与
模板批量命名、单独直接下载、统一直接下载或按文件族分目录的 ZIP 导出。源文件不会被伪装成处理结果。

输出注册表不复制 Blob、不写 storage、不持久化 object URL；关闭页面即释放。注册表限制为 200 项、
总计 1 GB，统一 ZIP 输入限制为 512 MB，直接下载单次最多 10 项。结果预览只在弹窗存续期间创建
object URL；仅安全位图和最多 128 KiB 文本读取内容，PDF、XLSX 与压缩包只显示元信息。

## 能力状态

| 状态 | 产品含义 | UI 规则 |
|---|---|---|
| 可用 | 已实现、已测试且当前浏览器具备必要能力 | 可进入工具，说明预设和输出 |
| 有限制 | 已实现基础路径，但编解码或保真度取决于浏览器 | 可进入工具，同时显示具体限制 |
| 规划中 | 已确认产品价值，尚未满足正确性、性能或安全门槛 | 可查看说明，不渲染伪操作按钮 |

任何工具不得仅凭扩展名宣称可处理。运行时能力检测失败时必须保留源文件并给出逐文件原因。

## 文件族与阶段路线

### 图片与 GIF

- 识别：JPG、JPEG、PNG、WebP、AVIF、GIF、SVG、BMP、TIFF、HEIC、HEIF、ICO。
- 已有基础：PNG/JPEG/WebP 转换、质量与尺寸、0°/90°/180°/270° 旋转、水平/垂直翻转、
  队列图片信息、批量命名、元数据随 Canvas 重编码清除、静态图片合成 GIF。
- 第一阶段推荐入口：转换格式、压缩图片、调整尺寸、查看信息、清除元数据、图片转 Base64、
  多图合成 GIF。
- 后续独立工具：裁剪、图片拼接、Favicon 生成、批量水印；GIF 拆帧、压缩、
  调速、裁剪、尺寸、视频和其他动图格式。
- HEIC/HEIF 已提供浏览器原生解码与按需加载的本地 WASM 后备路径，输入上限 64 MB；TIFF、ICO
  和部分 AVIF/BMP 仍只保证识别或取决于浏览器。任何格式都不得静默上传作降级方案。

### PDF

- 识别 PDF，并在独立 Tab 推荐合并、拆分、删除/提取/重排/旋转页面、PDF 转图片和信息查看。
- 第一阶段使用按需加载的 `pdf-lib` 1.17.1 完整解析页面树，开放队列顺序合并、逐页拆分、
  页码范围提取/删除、完整页码顺序重排和选中页 90°/180°/270° 旋转；每次都输出新文件。
- 资源预算为最多 20 份、单份 32 MB、合并输入 128 MB、总页数 500；逐页 ZIP 另限制为
  50 页和 256 MB 生成结果。加密/密码 PDF 拒绝处理，源文件始终保持不变。
- 轻量扫描继续提供版本、对象、线性化和元数据提示，但 UI 的页数来自完整解析器，不再使用估算值。
- 页面复制不是安全净化：批注、链接和页面动作可能保留；数字签名、表单、目录和文档级附件
  可能失效或未被复制。用户必须保留源文件并复核输出。
- 页面缩略图、拖拽式可视排序和 PDF 转图片仍为规划中；它们需要独立渲染器、像素内存预算、
  取消策略和恶意输入回归，不能用页面改写库冒充渲染能力。

### 文本与表格数据

- 识别 TXT、Markdown、HTML、JSON、YAML、XML、CSV、TSV、XLSX 等常见容器文件名。
- 已有 TXT/Markdown/HTML 与常用标记格式的批量结构转换。
- 已有独立表格数据工作台：CSV/TSV 可转 CSV、TSV、JSON、无宏单表 XLSX；XLSX 可读取工作表并
  转 CSV、TSV、JSON。它只读取值与公式缓存值，不执行公式、宏或外部链接，也不承诺保留样式。
- CSV/TSV 默认防御公式注入；数据预算为 CSV/TSV 16 MB、XLSX 32 MB、20,000 行、256 列、
  250,000 单元格。JSON/YAML/XML 的解析、统计、对比和编码转换仍保持规划中。

### 压缩包

- 首批在独立 Tab 识别 ZIP、读取中央目录并展示文件清单；其他压缩格式在具备可审计解析器后逐项加入。
- ZIP 提取只接受普通且不重名的文件路径，拒绝绝对路径、`..` 穿越、加密条目、未知压缩方式和
  超过预算的展开内容；支持 store 与浏览器具备 `DecompressionStream('deflate-raw')` 时的
  deflate，并在流式输出超过中央目录声明大小时立即中止。
- 查看内容、全部解压、选择性解压和文件列表必须防御路径穿越、重复路径、压缩炸弹、超大
  展开体积和隐藏文件误操作。
- 不以“浏览器可以读 ArrayBuffer”为由直接开放不受限解压。

## 常用预设

预设是可解释的参数组合，不是不可见的魔法：图片提供“网页照片”“透明素材”“社交分享”
和“隐私分享”；GIF 提供“表情循环”“步骤演示”；文本提供“README”“静态网页”“数据检查”。
用户选择预设后仍能看到并修改实际格式、质量、尺寸、时序或解析选项。

## 隐私与开发者信息红线

- 文件名、文件字节、像素、文本、解析结构和输出只存在当前浏览器内存，不写入 storage。
- 只有非内容偏好可使用 `toolbox.image-converter.*`；不使用 Cookie、账号、遥测或指纹。
- 不发起业务网络请求，不加载远端字体、编解码器或运行时资源；不支持的能力明确失败。
- 错误、测试 fixture、README 和交接文档不得包含真实用户内容、个人路径、私有邮箱、主机、
  端口、Token 或设备信息。
- HEIC 后备解码使用 `@discourse/heic` 1.0.0（Apache-2.0，libheif/libde265），WASM 原始体积约
  960 KB，仅在 HEIC/HEIF 输入时从同源静态资产加载；64 MB 输入与 8,000 万像素预算在解码前后生效。
- PDF 页面改写使用 `pdf-lib` 1.17.1（MIT），约 408 KB 原始体积 / 131 KB gzip，仅在进入 PDF
  工作台并导入文件后加载；不从 CDN 获取，不执行文档脚本，也不宣称输出已经净化。

## 性能与可靠性预算

- 首页单次最多接收 500 个文件、总计 2 GB；默认只读取每个文件最多 64 KiB 用于识别。
- 队列使用分组和渐进信息读取；单项图片预览只在弹窗打开时创建 object URL，并在关闭或卸载时释放。
- 结果队列只保存 Blob 引用，限制为 200 项 / 1 GB；统一 ZIP 生成前另检查 512 MB 输入上限，避免任务结果让页面内存无界增长。
- 像素处理继续执行现有最长边和总像素上限；长任务应支持逐项进度与停止。
- XLSX 复用一次读取的受限 ZIP 目录与字节缓冲，不重复加载整个容器；工作簿、关系、共享字符串和
  工作表 XML 另设条目上限，拒绝实体、外部关系和超预算表格。
- PDF 改写当前受同步内存预算保护并按文档顺序处理；继续扩大文件或页数预算前必须先进入 worker、
  增加取消/进度与针对恶意输入的专项测试。压缩包和大型文本同样不得绕过现有上限。
- 新编解码依赖必须记录首包增量；非首页必需能力采用延迟加载，不能拖慢“识别与推荐”。

## 验收原则

1. 同一文件在中英文界面获得一致分类、能力状态和可理解的限制说明。
2. 用户未选择工具前不读取完整文件，也不自动执行转换。
3. 可用操作能把兼容文件带入对应工作台；不兼容文件保持在首页并解释原因。
4. 用户无需先经过首页，也能从文件族 Tab 手动选择文件和工具；两条入口使用同一能力状态。
5. 混合文件、多文件、文件夹、空文件、伪装扩展名、超限输入和未知格式均有确定行为。
6. light/dark、1440/390px、键盘操作、reduced motion、无横向溢出和无业务请求通过回归。
7. 生产构建验证真实 HEIC→PNG 与 CSV→XLSX→重新导入，不用仅识别扩展名的假测试代替处理链路。
8. PDF 生产回归使用解析器生成的真实多页文件，验证准确页数、合并输出与下载，不以伪造 `%PDF` 文件头代替。
